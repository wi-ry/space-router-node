01 What We Built

A two-step challenge-verify registration flow that proves a home node operator has staked at least 1,000 CTC on the Creditcoin blockchain. Changes span two repositories.

Home Node

gluwa/space-router-node

03 Step-by-Step Flow

Each step shows what happens, where in the code it lives, and which project owns it.

1

Home Node Starts Up

Loads the EVM private key from SR_NODE_PRIVATE_KEY env var (or reads from SR_NODE_KEY_PATH file). Creates a NodeIdentity that derives the EVM address and provides an EIP-191 signing method.

space-router-node / app/identity.py → load_identity()
app/identity.py class NodeIdentity:
  def __init__(self, private_key):
    self._account = Account.from_key(private_key)

  def sign_message(self, text):
    message = encode_defunct(text=text)
    return self._account.sign_message(message).signature.hex()
2

Request Challenge

The node sends its EVM address to the Coordination API to request a registration challenge. This is the first HTTP call.

space-router-node / app/registration.py → register_node()
POST /nodes/register/challenge {
  "address": "0xabc...def"
}
3

On-Chain Stake Verification

The Coordination API queries the Creditcoin staking contract at 0x5d07...40Fd8 via stakeInfoEssential(address). Returns (stakedAmount, firstStakeTimestamp, isLocked). If stake < 1,000 CTC, the request is rejected with HTTP 403.

coordination-api / app/routers/registration.py → _get_staked_wei()
ABI stakeInfoEssential(address) → (uint128 stakedAmount, uint64 firstStakeTimestamp, bool isLocked)

// Minimum: 1000 CTC = 1000 * 10^18 wei
4

Issue Nonce

If stake is sufficient, the API generates a cryptographically random 64-char hex nonce, stores it in memory with a 5-minute TTL keyed by address, and returns it.

coordination-api / app/routers/registration.py → request_challenge()
Response: 200 {
  "nonce": "a1b2c3...64 hex chars",
  "expires_in": 300
}
5

Sign Nonce (EIP-191)

The home node signs the nonce using its private key with EIP-191 personal_sign. This creates a 65-byte signature that proves key ownership without revealing the private key.

space-router-node / app/identity.py → NodeIdentity.sign_message()
EIP-191 Signing # Prepends "\x19Ethereum Signed Message:\n{len}" before hashing
message = encode_defunct(text=nonce)
signature = account.sign_message(message)
6

Submit Verification

The node sends the signed nonce along with its endpoint URL, public IP, connectivity type, and optional metadata (region, label).

space-router-node / app/registration.py → register_node()
POST /nodes/register/verify {
  "address": "0xabc...def",
  "endpoint_url": "https://1.2.3.4:9090",
  "signed_nonce": "0x1a2b3c...",
  "public_ip": "1.2.3.4",
  "connectivity_type": "direct" | "upnp",
  "region": "us-west",
  "label": "my-node"
}
7

Verify Signature + Security Checks

The API performs four checks:
a) Recovers the signer via ecrecover and checks it matches the claimed address.
b) Consumes the nonce (one-time use, prevents replay).
c) Sybil prevention: rejects if another online node already uses the same public IP.
d) Classifies the IP via ipinfo.io (residential/datacenter/mobile).

coordination-api / app/routers/registration.py → verify_and_register()
8

Upsert Node Record

The API either creates a new node row or updates an existing one (matched by EVM address). Sets status to online with health_score: 1.0. Returns the node_id.

coordination-api / app/routers/registration.py → verify_and_register()
Response: 200 {
  "status": "registered" | "updated",
  "node_id": "uuid-here",
  "address": "0xabc...def",
  "endpoint_url": "https://1.2.3.4:9090",
  "ip_type": "residential",
  "ip_region": "Portland, US"
}
9

Start TLS Proxy Server

With the node_id in hand, the home node starts its TLS server (auto-generated self-signed cert) and begins accepting proxied traffic from the Proxy Gateway. On shutdown, it deregisters (best-effort PATCH to set status = offline).

space-router-node / app/main.py → _run()

04 Security Mechanisms

Multiple layers protect the network from unauthorized nodes and attacks.

🔒

EIP-191 Signatures

Nodes prove private key ownership without transmitting the key. The personal_sign prefix prevents cross-protocol replay attacks.

⛓

On-Chain Stake Gate

Must have ≥ 1,000 CTC staked on Creditcoin L1. Checked via stakeInfoEssential() before any nonce is issued. Economic cost deters Sybil attacks.

🛡

Sybil Prevention

Only one online node per public IP address. Different wallets attempting to register with the same IP get HTTP 409. Same wallet can re-register (upsert).

⏱

Nonce TTL + One-Time Use

Challenge nonces expire after 5 minutes and are consumed on first use. Prevents replay of old signatures.

🔍

IP Classification

Each node's IP is classified via ipinfo.io as residential, datacenter, mobile, or business. Datacenter IPs can be flagged or deprioritized.

🔐

TLS Everywhere

Home nodes auto-generate self-signed TLS certificates. All proxy traffic between the Gateway and Home Node is encrypted.

05 What Each Project Does

Side-by-side comparison of responsibilities.

🏠 space-router-node

Home Node Daemon
Owns:
• EVM private key storage + loading
• EIP-191 message signing
• HTTP calls to Coordination API
• TLS certificate generation
• TCP proxy server (CONNECT + HTTP forward)
• UPnP port mapping
• Graceful shutdown + deregistration

Does NOT:
• Verify signatures
• Query the blockchain
• Store any state in a database
• Know about other nodes

📡 coordination-api

Coordination API (FastAPI)
Owns:
• On-chain stake verification (web3.py)
• Nonce generation + TTL management
• EIP-191 signature recovery (ecrecover)
• Sybil prevention (unique IP check)
• IP classification (ipinfo.io)
• Node registry (SQLite / Supabase)
• Node routing for the Proxy Gateway

Does NOT:
• Hold any private keys
• Proxy any user traffic
• Generate TLS certificates

07 Error Scenarios

What happens when things go wrong at each step.

Scenario HTTP Code Who Handles Node Behavior
Stake below 1,000 CTC 403 Coordination API Raises RuntimeError, aborts startup
Invalid EVM address 400 Coordination API HTTPStatusError, aborts
Wrong signature (key mismatch) 403 Coordination API HTTPStatusError, aborts
Nonce expired (>5 min) 400 Coordination API HTTPStatusError, aborts
Nonce already used (replay) 400 Coordination API HTTPStatusError, aborts
Duplicate IP (Sybil) 409 Coordination API HTTPStatusError, aborts
Creditcoin RPC down 502 Coordination API HTTPStatusError, aborts
Deregistration fails any Home Node Logged as warning, NOT raised (best-effort)
